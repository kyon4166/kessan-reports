## 1. 目的・背景

これまでの2段階手作業ワークフロー
1. Claude AIプロジェクトで決算短信PDF → HTML生成（手動コピペ）
2. `kessan-app.onrender.com`（Render.com Webアプリ）にHTMLをアップロード → Googleスプレッドシートに反映

を、**Claude Code Webセッション内で完結する一本化パイプライン**に置き換える。

### 一本化の動機
- Render無料プランはスリープから起動に30秒要する
- 不具合発生時にRender側のコード修正が必要で二度手間
- Claude Code環境なら PDF→HTML→Sheets→GitHub Pages を全自動で一本化可能

### 2.2 ファイル名規約

`{証券コード4桁}_{決算期末西暦4桁}_{Q1|Q2|Q3|Q4}.html`

- 年度 = 決算期末の西暦4桁（2026年3月期 → 2026）
- Q = 通期決算は Q4、第1〜3四半期は Q1/Q2/Q3
- 例：`8725_2026_Q4.html`（MS&AD 2026年3月期 本決算）

### 2.3 スキップ判定（API課金ゼロ）

- PDFファイル名の先頭4桁＝証券コード とみなす
- repo 内に `{code}_*.html` が存在すれば、その PDF は処理済みとして SKIP
- 別期の同社処理が同時に走らない前提（年4回・期ごとに1回しか処理しない）

## 5. 解決したエラーとその対処

| 症状 | 原因 | 対処 |
|---|---|---|
| `cryptography` モジュール競合 | system debian版 vs pip版 | `pip install --upgrade cryptography`（警告は出るが動作） |
| SSL証明書検証エラー | `httplib2` が `REQUESTS_CA_BUNDLE` env var を見ない | `httplib2.Http(ca_certs="/etc/ssl/certs/ca-certificates.crt")` を `AuthorizedHttp` に明示的に渡す |
| P列 `#ERROR`（summary列） | 先頭が `+` の文字列（例：`"+14.9%増収..."`）が USER_ENTERED で数式扱いされる | `_safe_text()` を追加。先頭 `+`/`-`/`=`/`@` をシングルクォート escape。既存行は手動で修正 |
| Drive MCP `download_file_content` のtoken超過 | 544K char返却 | ファイルに保存 → Python base64 でデコード |
| Read tool で PDF 読めず | `pdftoppm` 未インストール → poppler-utils 入れても改善せず | Drive MCP `read_file_content` でテキスト抽出に切替 |
| 誤ったJSONをアップロード | 別アカウントのキー（yahoo-sheets-importer） | 正しい `portfolio-writer` のキーを再アップロード |

## 7. 将来計画

### 7.1 A案（推奨／第一目標）：Routines + Apple Shortcuts による自動化

**ゴール：** Mac / iPhone のショートカットアプリから1タップで「実行」を発火し、Claude Code に処理を委ねる。

#### 構成

```
[Mac / iPhone Shortcut]
       │
       │ HTTP POST  (Claude Code Routines API)
       ▼
[Claude Code Routine]
       │  Trigger: API call
       │  Action:  「実行」プロンプトを当該リポジトリの新規セッションで起動
       ▼
[Claude Code Web セッション]
       │  本ドキュメントの section 6 の手順を自動実行
       ▼
[GitHub main / Google Spreadsheet 更新]
       │
       ▼
[Shortcut に完了通知（任意）]
```

#### 必要な準備

1. **`tools/sheets.py` を main にマージ**
   - 現状は feature branch のみ。Routine から起動された新規セッションでも参照できるよう main に取り込む必要あり

2. **`GOOGLE_CREDENTIALS` 環境変数を Claude Code 環境設定に登録**
   - 現状は `/tmp/gcreds.json` を毎セッション Drive から取得している
   - Routine で自動起動される場合、Drive取得ステップを毎回挟むより環境変数化したほうが堅牢
   - Service Account JSON を1行化して `GOOGLE_CREDENTIALS` env var に保存
   - `sheets.py` は既に `GOOGLE_CREDENTIALS` env var をフォールバックでサポート済

3. **Routine 定義**
   - Trigger: API 呼び出し
   - Repository: `kyon4166/kessan-reports`
   - Branch: `main`
   - Prompt: 「実行」

4. **Apple Shortcut 作成**
   - Mac版／iPhone版それぞれ
   - アクション：URL内容取得（POST）→ Routines エンドポイント
   - 認証ヘッダ：Anthropic API キー
   - ホーム画面に追加して1タップ実行
   - iPhone からも同様に動作

#### A案のメリット
- Mac / iPhone のどちらからでも同一ボタンで起動
- ショートカットから実行しても、Webセッションで「実行」と打っても、どちらでも動作するため柔軟
- 手動セッションが空いている時はそちらで処理してもよい

#### A案の懸念点
- Anthropic API キーの管理（Shortcut内に保存される）
- Routine 実行のトークンコスト（Sonnet使用前提でも数十件は制限に当たり得る）
- Routine 失敗時のリトライ／通知設計
